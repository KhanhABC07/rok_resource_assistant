# CHR-001 — Switch Characters

Use together with `00_SHARED_CONTEXT.md`. Implement only this task.

## Codex Prompt

```text
Implement verified character switching for up to twelve characters per account. Add character slot, display fingerprint, optional kingdom ID, and verification metadata. Build a CharacterSwitchWorkflow that verifies the active account, opens character management, supports paging/scrolling, locates the target using semantic templates and/or OCR fingerprint, selects it, confirms, waits for reload, and verifies the active character before updating instance_session. Do not treat an in-memory cache as proof. Add replay tests for already active, target on later page, duplicate names with different kingdom fingerprints, missing target, and reload timeout.
```
